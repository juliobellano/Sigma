## What is Sigma?

Sigma is a real-time, multimodal AI cooking assistant that lives in your browser. Your hands are covered in flour — you can't type. Your eyes are on the pan — you can't read a recipe. Sigma listens, watches, and responds with **voice + vision**, no screen touching required.

Ask it to set a timer. Ask it where the fish sauce is. Ask it what to substitute for lemongrass. Ask it to show you how to julienne carrots. Sigma sees your kitchen through your camera and talks back in under a second — exactly like a chef standing next to you.

**This is not a chatbot. There is no text box. Sigma is entirely hands-free.**

## Features

### Real-Time Voice Interaction
- Bidirectional audio via Gemini Live API (`gemini-2.5-flash-native-audio-preview-12-2025`)
- 16 kHz PCM microphone input via AudioWorklet — zero latency
- 24 kHz PCM speaker output with ring-buffer smoothing
- Barge-in detection: interrupt Sigma mid-sentence naturally
- Session resumption: conversation context survives page refresh
- Both input and output transcribed in real-time

### Live Camera Vision
- Continuous camera feed captured at 1 FPS, 768×768 px JPEG
- Every frame streamed directly to Gemini for context
- Sigma proactively warns about hazards it sees (smoke, overflowing pot, wrong pan)
- Flash animation confirms each frame capture

### 8 Agentic Tools
Sigma calls these tools autonomously based on conversation context:

| Tool | Trigger | What Happens |
|---|---|---|
| `set_timer` | "set a 5 minute timer for the eggs" | Countdown widget appears, TTS announces completion |
| `find_object` | "where's the garlic?" | Camera frame → Gemini's native bounding box detection → coordinates drawn on canvas as overlay |
| `find_substitute` | "I'm out of lemongrass" | Google Search grounding → available ingredients checked via camera |
| `how_to` | "how do I dice an onion?" | Camera frame → illustrated 4-step guide generated by image model |
| `set_goals` | YouTube tutorial loaded | Tutorial steps parsed and loaded into progress tracker |
| `check_current_step` | "what step am I on?" | Returns current step from tutorial state |
| `check_next_step` | "what's next?" | Previews the upcoming step |
| `next_step` | "I'm done with this" | Advances the tutorial step index |

### Multi-Model Pipeline
Five specialized Gemini models work in concert behind the scenes:

| Model | Role |
|---|---|
| `gemini-2.5-flash-native-audio-preview-12-2025` | Primary: voice conversation + tool calling + vision reasoning |
| `gemini-3-flash-preview` | Object localization using Gemini's native bounding box output — returns `[y, x, y, x]` coordinates on a 0–1000 scale, drawn onto the camera frame client-side |
| `gemini-3.1-flash-image-preview` | Generate illustrated 4-step how-to guides from the live camera frame |
| `gemini-2.5-flash-preview-tts` | TTS announcements for timer completions |
| `gemini-3.1-pro-preview` | YouTube cooking tutorial step extraction |

### Tutorial Mode
Paste a YouTube cooking video URL at the splash screen. Sigma sends the video to Gemini, extracts all steps as structured JSON, and loads them into a visual progress bar. Navigate steps hands-free with voice commands.

### Sigma's Personality
Sigma is designed to feel like a best friend who cooks. The system instruction gives it a distinct character:
- **Brief**: 1–2 sentences max — you're cooking, not reading
- **Warm**: "Nice knife work!" / "No worries, we can fix that"
- **Proactive**: warns about safety hazards without being asked
- **Grounded**: uses tool results, not hallucinations, for factual answers

## Tool Call Lifecycle

Here is exactly what happens when you say *"Where is the salt?"*:

```
1. Audio chunk (16kHz PCM) → Gemini Live API
2. Gemini transcribes: "Where is the salt?"
3. Gemini decides: call find_object("salt")
4. App.jsx receives toolCall event
5. Immediately responds to Gemini: "Searching, one sec…"
6. Captures full-resolution camera frame
7. Sends frame + "salt" to gemini-3-flash-preview
8. Parses bounding boxes: [{label:"salt", box_2d:[y,x,y,x]}]
9. Draws green boxes on canvas → base64 JPEG
10. Displays BBoxWidget in empty slot
11. Enqueues result: {type:"find_object", found:true, label:"salt"}
12. On next turnComplete → sendClientContent:
    "[BACKGROUND_RESULT] Found salt — highlighted on screen."
13. Gemini announces naturally: "There's the salt — right by your cutting board!"
14. BBoxWidget auto-dismisses after 15 seconds
```

No database. No round-trips to a server. The entire pipeline runs in ~1–2 seconds.

## Getting Started

### Prerequisites
- Node.js 18+
- A Google AI Studio API key
- A modern browser with WebRTC support (Chrome recommended)
- A webcam and microphone

### Setup

```bash
cd frontend
npm install
```

Create `frontend/.env`:
```
VITE_GEMINI_API_KEY=your_google_ai_studio_key_here
```

```bash
npm run dev
# → http://localhost:5173
```

Open the app, grant microphone and camera permissions, and start talking.

### Using Tutorial Mode

On the splash screen, paste a YouTube cooking tutorial URL before starting the session. Sigma will analyze the video with Gemini, extract all steps, and load them into the step progress bar. Navigate with voice: *"next step"*, *"what step am I on?"*, *"what's coming up?"*

## Google Cloud Integration

The frontend connects directly to Gemini via the `@google/genai` JavaScript SDK using a Google AI Studio API key. There is no backend server — every API call originates in the browser.

Google Search grounding is used for the `find_substitute` tool — a Google Cloud-native capability that lets Gemini query the web for real ingredient substitutes before cross-referencing the camera feed.

## What I Learned

**Real-time audio is unforgiving.** The Gemini Live API expects 16 kHz PCM chunks delivered consistently. Getting this right required AudioWorklet processors running in a dedicated audio thread, a ring buffer for smooth 24 kHz playback, and careful Float32 ↔ Int16 conversion on both ends.

**Background tasks + result injection is the key pattern.** Tool calls like `find_object` are computationally slow (they spin up a second model). The trick is to respond immediately to Gemini with a placeholder ("searching…"), run the heavy task in the background, then inject the result back as `[BACKGROUND_RESULT]` on the next turn. This keeps the conversation flowing without blocking.

**Context window compression matters at scale.** Long cooking sessions accumulate a lot of context. The `SlidingWindow` compression config prevents the session from degrading over time — essential for a tool meant to be used for 30-60 minutes continuously.

**Multi-model orchestration requires clear handoffs.** Each secondary model has a different output format (bounding boxes, JPEG images, PCM audio, JSON arrays). Building a clean tool dispatch layer in `App.jsx` with a result queue prevents race conditions between concurrent tool calls.
